\# Dataset Setup Instructions  
\#\# Overview  
This project uses a subset of the Google Cluster Trace dataset.   
The dataset is not included in the repository due to its large size and must be downloaded separately.

\#\# 1\. Download Dataset  
Download the dataset from Kaggle:  
https://www.kaggle.com/datasets/derrickmwiti/google-2019-cluster-sample

Steps:  
1\. Open the link above  
2\. Click \*\*Download\*\*  
3\. Extract the downloaded \`.zip\` file

\#\# 2\. Place Dataset in Project  
After extraction, locate the CSV file and place it in the following directory:  
\`\`\`plaintext  
masters-project/  
 subset/  
   borg\_traces\_data.csv

\#\# 3\. Install Required Dependencies  
Make sure you have Python installed.  
\*\*Environment used in this project:\*\*  
\- Python 3.12.4 (Conda base environment)  
Install the required packages:  
\`\`\`bash  
pip install pandas pyarrow

\#\# 4\. (Optional) Install Parquet Viewer in VS Code  
To view \`.parquet\` files in a table format inside VS Code, install one of the following:  
\#\#\# Recommended extensions  
\- Parquet Visualizer (Lucien Martijn)  
These extensions allow parquet files to be opened in a tabular format instead of raw binary.

\#\# 5\. Running the Notebook  
The notebook supports both \*\*local\*\* and \*\*Google Colab\*\* environments.

\#\#\# To run locally  
1\. Open the project root (\`masters-project/\`) in VS Code or similar IDE  
2\. Ensure \`borg\_traces\_data.csv\` is inside \`subset/\`    
3\. Open:

   \`\`\`plaintext  
   subset/subsetv1.ipynb  
4\. Select a Python environment, for example:

- base (Python 3.12.4)

5\. Click Run All

\#\# 6\. Expected Outputs  
After running the notebook, parquet files will be generated. For example:  
\`\`\`plaintext  
subset/  
  job\_metrics\_v0.parquet  
  job\_profiles\_confidence\_v0.parquet

\#\# 7\. Notes  
\- Dataset files (\`.csv\`) and generated outputs (\`.parquet\`) are excluded from GitHub via \`.gitignore\`  
\- All outputs can be regenerated by running the notebook  
\- This setup ensures reproducibility across different environments
